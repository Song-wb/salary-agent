# 行业薪酬洞察AI智能体 - 技术方案文档

## 一、项目概述

### 1.1 项目背景

开发一款智能体，从主流招聘网站公开抓取岗位薪资信息，自动汇总分析并生成行业薪资对比报告，支撑企业薪酬决策。

### 1.2 项目目标

1. 抓取BOSS直聘、智联招聘等平台指定行业/地区的公开招聘信息
2. 按岗位、经验、学历等维度统计薪资数据
3. 生成可视化薪资对比报告，支撑企业薪酬决策

**分析维度**：

| 维度   | 分析内容     | 统计指标               |
| ---- | -------- | ------------------ |
| 岗位维度 | 各岗位薪资分布  | 均值、中位数、分位值         |
| 经验维度 | 不同经验薪资曲线 | 0-1年、1-3年、3-5年、5年+ |
| 学历维度 | 不同学历薪资对比 | 大专、本科、硕士、博士        |
| 地区维度 | 不同城市薪资对比 | 一线、新一线、二线城市        |
| 时间维度 | 薪资变化趋势   | 周/月/季度变化           |

**统计指标**：

- 25分位值（Q1）：较低端薪资水平
- 50分位值（Q2/中位数）：市场平均水平
- 75分位值（Q3）：较高端薪资水平
- 均值：整体平均薪资
- 标准差：薪资离散程度
- 岗位数量：各维度岗位供给量

## 五、AI赋能方案

### 5.1 基线方案

**提示词设计**：

```
你是一位专业的薪酬分析专家，擅长分析招聘市场数据并生成行业薪酬报告。

角色：资深人力资源数据分析专家
任务：分析招聘薪资数据，生成专业的薪酬分析报告
约束：分析要客观、数据要准确、报告要专业

数据输入：
行业：{industry}
地区：{region}
时间范围：{time_range}
样本数量：{sample_count}

分析维度：
1. 岗位分布统计
2. 薪资区间分布
3. 经验要求与薪资关系
4. 学历要求与薪资关系
5. 供需趋势分析

请按照以下结构生成分析报告：
一、数据概览
二、薪资分布分析
三、岗位薪资对比
四、经验-薪资曲线
五、学历-薪资关系
六、行业趋势预测
七、结论与建议
```

### 5.2 增强方案

#### 5.2.1 Few-shot示例学习

```
示例1：薪资分布分析
"根据采集的1,256条数据显示，Java开发岗位的薪资呈现明显的偏态分布。25分位薪资为15K，50分位为22K，75分位为32K。其中，30-40K薪资区间的岗位占比最高，达到35%，表明市场对中高级Java开发人员需求旺盛。"

示例2：经验-薪资分析
"数据分析显示，薪资随工作经验呈非线性增长趋势。0-1年经验岗位平均薪资12K，1-3年经验跃升至18K（增长50%），3-5年经验达到25K（增长39%），5年以上经验趋于平稳，平均薪资约32K。"
```

#### 5.2.2 RAG检索增强

**知识库构建流程**：

```
┌─────────────────────────────────────┐
│            知识库构建流程            │
├─────────────────────────────────────┤
│ 1. 数据收集 → 历史薪酬报告、行业报告 │
│ 2. 文本处理 → 提取关键指标和结论    │
│ 3. 向量化 → Embedding生成向量       │
│ 4. 存储 → FAISS向量数据库           │
│ 5. 检索 → 查询时匹配相关历史数据    │
└─────────────────────────────────────┘
```

**检索应用场景**：

- 对比分析：将当前数据与历史数据对比，发现变化趋势
- 行业对标：参考同行业其他公司的薪酬水平
- 异常检测：识别薪资数据中的异常值和不合理数据

#### 5.2.3 Agent工作流设计

**Agent决策流程**：

```
┌─────────────────────────────────────┐
│           Agent决策流程             │
├─────────────────────────────────────┤
│  用户请求                           │
│      │                             │
│      ▼                             │
│  [解析需求] → 需要抓取数据？         │
│      │                             │
│      ├── Yes → [启动爬虫任务]       │
│      │         │                   │
│      │         ▼                   │
│      │    [数据入库] → [AI分析]     │
│      │                             │
│      └── No  → [查询历史数据]       │
│                │                   │
│                ▼                   │
│           [AI分析] → [生成报告]     │
└─────────────────────────────────────┘
```

**工具集设计**：

| 工具名称 | 功能说明        | 参数                                      |
| ---- | ----------- | --------------------------------------- |
| 启动爬虫 | 按条件抓取招聘网站数据 | industry, region, position, date\_range |
| 查询数据 | 从数据库查询历史数据  | industry, region, filters               |
| 统计分析 | 执行薪资统计计算    | data, dimensions                        |
| 生成图表 | 生成可视化图表     | data, chart\_type                       |
| 生成报告 | 生成Doc/PDF报告 | analysis\_result, template              |

#### 5.2.4 智能数据清洗

利用大模型进行智能数据清洗：

- **薪资格式统一**：自动识别"15-25K"、"15000-25000"等不同格式并统一
- **异常值检测**：识别明显不合理的薪资数据（如过高或过低）
- **数据补全**：对缺失字段进行智能推断和补全
- **重复数据去重**：识别并移除重复的招聘信息

#### 5.2.5 自然语言交互

支持自然语言查询和对话式交互：

- 支持用户用自然语言提问，如"北京Java开发的薪资水平如何？"
- 自动解析问题中的行业、地区、岗位等参数
- 生成自然语言的分析回答和建议

## 六、风险评估与应对策略

### 6.1 技术风险

| 风险描述 | 风险等级 | 应对策略 |
| -------- | -------- | -------- |
| 大模型输出不稳定，可能产生不准确或不一致的分析结果 | 高 | 建立输出校验机制，对关键指标进行二次验证；设置置信度阈值，低于阈值时触发人工复核 |
| 特定场景识别准确率低，如特殊薪资格式、行业术语理解偏差 | 中 | 积累行业特定语料库；引入Few-shot示例学习；建立错误反馈机制持续优化 |
| 模型幻觉问题，可能生成看似合理但实际错误的数据解读 | 高 | 要求模型基于实际数据生成分析；添加数据溯源功能；关键结论需标注数据来源 |
| 大模型API调用失败或响应超时 | 中 | 实现API调用重试机制；建立降级策略，使用本地规则引擎替代；设置超时中断并提示用户 |

### 6.2 数据风险

| 风险描述 | 风险等级 | 应对策略 |
| -------- | -------- | -------- |
| 训练/测试数据不足，统计分析结果缺乏代表性 | 高 | 扩大数据源覆盖范围；设置最低样本量阈值；使用统计方法评估数据置信度 |
| 抓取数据质量不高，存在大量无效或错误数据 | 中 | 建立多级数据清洗流程；设置数据质量评分机制；定期抽检数据准确性 |
| 薪资数据缺失严重，关键维度分析受限 | 中 | 实现智能缺失值填补；降低缺失比例要求；提供多维度组合查询 |
| 数据更新不及时，报告反映的是过时市场状况 | 低 | 建立自动化数据更新机制；报告需标注数据时间范围；提醒用户定期刷新数据 |
| 反爬策略导致数据抓取不完整或被封禁 | 高 | 实施多代理IP轮换；控制请求频率和行为模拟；准备多个备用数据源 |

### 6.3 协作风险

| 风险描述 | 风险等级 | 应对策略 |
| -------- | -------- | -------- |
| 项目进度延迟，无法按时交付 | 高 | 制定详细项目计划；设置关键里程碑检查点；预留应急缓冲时间 |
| 团队成员技术能力不足，影响开发效率 | 中 | 组织技术培训；建立代码审查机制；必要时引入外部技术支持 |
| 接口对接问题，前后端联调困难 | 中 | 提前定义清晰API接口规范；采用Swagger进行接口文档管理；安排专门联调时间 |
| 需求变更频繁，导致返工和进度延误 | 中 | 建立需求变更管理流程；采用敏捷开发迭代；控制变更范围和频率 |

### 7.2 数据清洗规则

**薪资数据处理**：

| 原始格式        | 统一格式        | 处理逻辑          |
| ----------- | ----------- | ------------- |
| 15-25K      | 15000-25000 | 去除K，乘以1000    |
| 15000-25000 | 15000-25000 | 直接保留          |
| 1.5-2.5万    | 15000-25000 | 去除"万"，乘以10000 |
| 面议          | NULL        | 标记为缺失         |

**异常值检测规则**：

- 薪资下限 < 2000：标记为异常
- 薪资上限 > 200000：标记为异常
- 薪资范围跨度 > 50000：标记为异常

### 7.4 数据更新策略

**增量更新**：

- 每日定时抓取新增数据
- 按发布时间筛选，只抓取当日/近几日数据
- 避免重复抓取历史数据

**全量更新**：

- 每周/每月执行一次全量抓取
- 更新历史数据，确保数据完整性
- 重新计算统计指标
